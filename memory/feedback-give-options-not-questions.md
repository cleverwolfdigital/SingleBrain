When a decision comes up, present **2–3 concrete options** (ideally with a recommended default marked) instead of asking an open-ended question. Quincy would rather pick from choices than answer from scratch.

**Why:** He's the decision-maker on a fast-moving solo operation; options are faster to act on and reduce back-and-forth.

**How to apply:** Frame forks as "Option A / B / C" with a one-line tradeoff each and a recommendation. Still surface genuine conflicts (see the SingleBrain reconcile pattern), but resolve them into options, not questions. Reserve true open questions for cases where no reasonable option set can be generated (e.g. a hard unknown like final pricing only he knows).
