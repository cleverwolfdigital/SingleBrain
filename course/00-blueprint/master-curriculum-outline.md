## 2. Curriculum Structure (v0.9 — restructured around the decided direction)

### Module 0 — Orientation: The Loop
- 0.1 The promise, the proof (this course's own build receipts), and how the course works
- 0.2 **The Loop** (signature framework): compounding builders run on closed loops — session loops, launch loops, review loops. Stuck builders run on open-ended effort. Everything in this course is a loop.

### Module 1 — Think Like an Operator (the mindset before the mechanics)
- 1.1 **The Map: How AI Actually Knows Things** ⏳ — frozen training vs. the context window (the only door in). Where the buzzwords live: prompt engineering → context engineering → personal context management. Asset: context-engineering-map.svg. Defines Single Source of Truth (SSOT): AI reads FROM your truth; it is never itself your truth.
- 1.2 **Don't Read Faster — Recontextualize** — the AI-era reading skill: command AI to reshape output BEFORE reading ("4 lines," "dumb it down," "decisions only"). The Read/Skim/Skip rule. You are the decision-maker, not the proofreader. *(Cross-listed with Digital Speed Reading Course — see Section 6.)*
- 1.3 **You Can't Break It** — beginner confidence lesson: versioning, backups, change logs mean every mistake is reversible. Nothing is ever silently lost.

### Module 2 — Build Your Blueprint (setup)
- 2.1 **Own Your Source of Truth** — the 4-6 documents that define your project (outline, audience, voice, frameworks, source log) and why they live in files YOU own, outside any AI tool
- 2.2 **Set Up Your Workspace** ⏳ — AI project setup: upload the stable reference layer only; write tight project instructions (the highest-leverage 5 minutes)
- 2.3 **One Conversation, One Deliverable** — why focused sessions beat mega-threads; naming conventions
- 2.4 Action project: create your blueprint documents from a fill-in template

### Module 3 — Run the Loop (production)
- 3.1 **The Session Loop** ⏳ — the four-move, five-minute routine: save → file & update → re-upload → read the briefing. No session starts from zero.
- 3.2 **The Librarian** ⏳ — automating the tedious half with ready-made prompts (asset: claude-code-session-prompts.md, adapted for non-technical users with a manual fallback version)
- 3.3 **The Stale Blueprint Problem** — the four ways your AI's context silently degrades (stale blueprint, dilution, contradiction drift, orphaned context) and the loop habits that prevent each (PCM framing credited to Forte — S-003)
- 3.4 Action project: run three full production sessions on your own project

### Module 4 — Build on Giants (working with other people's ideas)
- 4.1 **Borrowing with Integrity** — the Source & Attribution Log; extract principles, never expression; the transformation test; why citing sources adds authority
- 4.2 **Future-Proof Your Work** — the perishability convention (⏳): principles age slowly, buttons age fast; quarterly review loop
- 4.3 Action project: log and transform one source into an original piece

### Module 5 — Launch Loop (capstone) *— Hormozi-derived, see S-001*
- 5.1 **Ship the thing** — finish one real asset with your system (mini-course, content series, or offer)
- 5.2 **The Promoter Loop** — two-tier incentives, win-win-win rewards, copy-paste collateral, leaderboard feedback: get other people telling people about your thing
- 5.3 Action project: launch to your first audience, however small, with a mini promoter offer
